1인 개인 프로젝트

2024.02 - 2024.04 (기획 및 백엔드 개발)

동기식 이미지 업로드 시 발생하는 서버 병목 현상과 클라이언트 대기 시간을 줄이기 위해, **NestJS**와 메시지 큐 라이브러리인 **BullMQ**를 결합하여 설계한 고성능 비동기 이미지 처리 파이프라인입니다. 외부 클라우드 의존도를 낮추기 위해 **자체 리눅스 서버** 환경에서 백라이브러리를 최적화했습니다.

✓ BullMQ 기반 비동기 아키텍처: Redis 기반의 BullMQ 워커 시스템을 도입하여 업로드 요청과 리사이징/압축 처리를 분리, 사용자 응답 속도 개선.
✓ 이미지 인프라 최적화: Sharp 라이브러리를 연동해 원본 이미지를 WebP 포맷으로 자동 변환 및 압축을 수행하여 스토리지 공간 평균 60% 이상 절감.
✓ 자체 리눅스 서버 리소스 관리: 서버 파일 시스템 및 인프라를 직접 제어하며 동시 프로세싱 제한(Concurrency 제어) 설정을 통해 CPU 과부하 방지 안정성 확보.

웹소켓 기반 실시간 라이어 게임 플랫폼

1인 개인 프로젝트

2024.07 - 2024.09 (기획 및 풀스택 개발)

사용자들이 브라우저만으로 간편하게 모여 즐길 수 있는 실시간 멀티플레이어 보드게임 웹 서비스입니다. **Next.js**로 반응형 인터페이스를 구축하고, **NestJS** 웹소켓 게이트웨이와 **Redis** 인메모리 데이터베이스를 결합하여 지연 시간 없는 초고속 게임 상태 동기화를 구현했습니다.

✓ Redis 활용 고속 게임방 관리: 잦은 변경과 빠른 응답이 필요한 게임 룸의 상태, 대기열(Queue), 라이어 선정 및 투표 데이터 등을 RDBMS 대신 Redis 세션 및 캐싱으로 관리하여 고성능 데이터 입출력 달성.
✓ WebSocket 실시간 양방향 통신: NestJS WebSocket(Socket.io) 게이트웨이를 설계하여 다수의 플레이어 간 실시간 채팅, 턴(Turn) 전환, 투표 집계 현황을 브라우저에 실시간 반영.
✓ Next.js 실시간 게임 UX 최적화: 상태 변화에 따른 컴포넌트 실시간 렌더링 최적화, 타이머 동기화 로직 분리를 통해 플레이어 간 데이터 싱크 오차 최소화.

RAG 기반 학사 정보 및 전공 지식 Q&A 챗봇

전공 연계 개인 프로젝트

2024.10 - 2024.12 (기획 및 파이프라인 개발)

자연어처리(NLP) 전공 수업을 수강하며 LLM의 고질적인 문제인 할루시네이션(환각 현상)에 의문을 느껴, 이를 해결하기 위한 **RAG(검색 증강 생성)** 아키텍처 기반의 지식 답변 챗봇을 설계했습니다. 학사 규정 및 내재화된 PDF 데이터를 벡터화하여 정확한 출처 기반의 답변을 제공하도록 파이프라인을 구축했습니다.

✓ 문서 전처리 및 Vector DB 구축: LangChain을 활용하여 복잡한 PDF/텍스트 도큐먼트를 Chunk 단위로 분할하고, OpenAI Embedding 모델을 통해 고차원 벡터로 변환 후 Vector DB(FAISS/Chroma)에 저장.
✓ 정확도 중심의 컨텍스트 검색: 사용자 프롬프트가 입력되면 Cosine Similarity 기반의 시맨틱 검색을 수행, 연관성이 가장 높은 문서(Context)를 프롬프트에 동적으로 주입하여 할루시네이션 방지.
✓ 스트리밍 답변 UI 최적화: LLM의 응답 지연 시간 동안의 사용자 경험(UX)을 개선하기 위해 Server-Sent Events(SSE) 방식을 도입해 답변 글자가 실시간으로 타이핑되는 듯한 스트리밍 인터페이스 구현.
